# Turbot v5 Style Guide

## Overview

### Capitalization

Turbot fields use [Lower Camel Case](http://wiki.c2.com/?LowerCamelCase).  
       Examples: type, properties, definitions.

In general, properties of cloud resources will match the cloud provider's property.

AWS properties use [Pascal case](http://wiki.c2.com/?PascalCase), i.e., upper case camel.  
       Examples: InstanceId, QueueName, Ec2Service.

### Common Properties

Use the `Aws` property, which currently contains the AWS account ID and region ID in each AWS resource definition. Even the AWS account and region definitions should contain this property.

### Testing

1. If no description is provided the input will be used. 
2. Valid tests can either use the input as the description or have more specific descriptions. Do not add any words about the test being valid to the description. An example is `min length`. 
3. Invalid tests \(i.e., `expected: false`\) must have a description that starts with `invalid -` . An example is `invalid - too long`.
